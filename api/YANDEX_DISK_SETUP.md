# Настройка Яндекс.Диска для хранения изображений

Данное руководство поможет настроить интеграцию с Яндекс.Диском для хранения изображений пользователей.

## Требования

- Аккаунт Яндекс
- Подключенный Яндекс.Диск

## Шаги настройки

### 1. Получение OAuth токена

1. Перейдите на страницу [Яндекс OAuth](https://oauth.yandex.ru/)
2. Нажмите "Зарегистрировать новое приложение"
3. Заполните форму:
   - **Название**: FOHOW Interactive Board (или любое другое)
   - **Платформы**: Выберите "Веб-сервисы"
   - **Redirect URI**: `https://oauth.yandex.ru/verification_code`
   - **Права доступа**: Выберите "Яндекс.Диск REST API" и отметьте:
     - Чтение всего Диска
     - Запись в любое место на Диске
     - Доступ в любое время (offline)

4. После создания приложения скопируйте **ID приложения**

5. Получите токен через URL (замените `CLIENT_ID` на ваш ID приложения):
   ```
   https://oauth.yandex.ru/authorize?response_type=token&client_id=CLIENT_ID
   ```

6. Авторизуйтесь и скопируйте полученный **access_token**

### 2. Создание базовой директории на Яндекс.Диске

1. Откройте [Яндекс.Диск](https://disk.yandex.ru/)
2. Создайте папку `/FOHOW` (или любое другое имя)
3. Эта папка будет использоваться как базовая директория для хранения всех файлов пользователей

### 3. Настройка переменных окружения

1. Создайте файл `.env` в папке `api/` (если его нет):
   ```bash
   cp .env.example .env
   ```

2. Откройте `.env` и добавьте/обновите следующие переменные:
   ```env
   # Настройки Яндекс.Диска для хранения изображений
   YANDEX_DISK_TOKEN=your_oauth_token_here
   YANDEX_DISK_BASE_DIR=/FOHOW
   ```

   Где:
   - `YANDEX_DISK_TOKEN` - OAuth токен, полученный на шаге 1
   - `YANDEX_DISK_BASE_DIR` - путь к базовой директории на Яндекс.Диске (созданной на шаге 2)

3. Сохраните файл

### 4. Проверка настройки

1. Запустите API сервер:
   ```bash
   cd api
   npm start
   ```

2. Проверьте логи при запуске. Вы должны увидеть:
   ```
   [Yandex Disk Service] Сервис успешно инициализирован
   [Yandex Disk Service] Базовая директория: /FOHOW
   ```

3. Если видите ошибку `[FATAL ERROR] Переменная окружения YANDEX_DISK_TOKEN не задана`, проверьте файл `.env`

### 5. Тестирование загрузки изображений

1. Войдите в приложение
2. Перейдите в раздел "Изображения"
3. Попробуйте загрузить изображение
4. Проверьте, что изображение появилось:
   - В списке изображений в приложении
   - На Яндекс.Диске в папке `/FOHOW/{userId}.{personalId}/library/`

## Структура хранения на Яндекс.Диске

```
/FOHOW/                              # Базовая директория (YANDEX_DISK_BASE_DIR)
├── {userId}.{personalId}/           # Папка пользователя
│   └── library/                     # Личная библиотека
│       ├── {filename1}.webp         # Изображения в корне
│       ├── {filename2}.webp
│       └── {folderName}/            # Пользовательские папки
│           └── {filename3}.webp
└── SHARED/                          # Общая библиотека
    ├── pending/                     # Ожидающие модерации
    └── {sharedFolderName}/          # Категории общей библиотеки
```

## Возможные ошибки и их решение

### Ошибка: "Ошибка авторизации на сервере хранилища"

**Причина**: Невалидный или истекший токен OAuth

**Решение**:
1. Проверьте, что токен правильно скопирован в `.env`
2. Получите новый токен (токены могут истекать)
3. Убедитесь, что при создании приложения были выбраны правильные права доступа

### Ошибка: "Недостаточно места на сервере хранилища"

**Причина**: Закончилось свободное место на Яндекс.Диске

**Решение**:
1. Проверьте объем занятого места на [Яндекс.Диске](https://disk.yandex.ru/)
2. Удалите ненужные файлы или расширьте хранилище

### Ошибка: "Не удалось подключиться к серверу хранилища"

**Причина**: Проблемы с сетевым подключением к API Яндекс.Диска

**Решение**:
1. Проверьте интернет-соединение на сервере
2. Убедитесь, что нет блокировок firewall для `https://cloud-api.yandex.net`
3. Проверьте логи сервера для детальной информации

### Сервер не запускается с ошибкой FATAL ERROR

**Причина**: Не установлены обязательные переменные окружения

**Решение**:
1. Убедитесь, что файл `.env` существует в папке `api/`
2. Проверьте, что в `.env` установлены `YANDEX_DISK_TOKEN` и `YANDEX_DISK_BASE_DIR`
3. Перезапустите сервер

## Безопасность

1. **Никогда не коммитьте файл `.env` в git** - он уже добавлен в `.gitignore`
2. **Храните токен в безопасности** - это ключ доступа к вашему Яндекс.Диску
3. **Регулярно обновляйте токены** - для повышения безопасности
4. **Используйте отдельный аккаунт Яндекс** для production окружения

## Мониторинг

Рекомендуется настроить мониторинг:

1. **Объем хранилища**: Регулярно проверяйте свободное место
2. **Логи ошибок**: Отслеживайте ошибки в логах сервера
3. **Квоты API**: Следите за использованием квот API Яндекс.Диска
